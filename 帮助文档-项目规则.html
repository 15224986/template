z-index 的使用规则

在做大型的项目中，由于页面多、结构复杂、功能多，如果z-index没有使用好，会造成后期调节层级比较麻烦，特别是需要兼容IE浏览器的情况下。
1.弹出框如：modal、Alert、Messager等（多为使用fixed定位），使用的z-index是999~1099之间。
2.弹出框如：tooltip、popover 、dropdown、select2、模拟的弹出框等（多为使用absolute定位），使用的z-index是800~900之间。
3.页面分为两部分，下部分是一个绝对定位的可拖动地图等，需要给上部分加z-index时，上部的z-index使用是：500~700之间。 下部分的z-index使用是：100~199以内，里面的浮动点、线等多为字默认图层（如果需要多为：9、19、29~99）。

公共方法的调用

以后所有的js方法都是用data-toggle=""这个进行公共调用，去除class类名的调用，class类名负责样式。

文件结构

所有项目的前端资源都存在于project文件夹
1.内部的具体项目名称（例如：百度-百度传课、网易-网易云课堂、腾讯视频等）
2.项目内部分为：文档-doc、设计图-psd、静态页-html
3.设计图-psd内部分为：images给app做的切片、web电脑端设计图、app设计图、WeChat微信设计图、admin网站后台设计图
4.html内部分为：web电脑端静态页、phone手机端静态页、admin网站后台静态页
开发目录结构为：
1.data 前端一些数据或者模拟后台返回的json
2.fonts 字体文件
3.files 模拟上传的文件，与页面无关
4.Images 图片文件
5.Images/icon 图标文件
6.scripts 自己写的js文件
7.vendor 插件库
8.styles 样式文件（sass存放scss文件、css存放css文件、sass/components存放的sass组件、*.css下载的css库）
9.public HTML 静态文件局部模板
10.XX模块-XX模块-XX模块-XXX-列表（详情、表单）.html

命名方式

参考 http://www.css88.com/archives/5361

根据不同文件不同用途选择不同命名形式
snake_case: 蛇形小写 用于连接单词
camelCase: 驼峰命名 用于js中方法名
PascalCase: 帕斯卡命名 用于js中类名、id名
dot.case: 点分命名 用于html文件中元素的name属性
dash-case: 线分命名 用于html、css、js、img文件名和css中的class名
separate words: 独立单词 用对js中对象的键或其他

通用原则

源文件和生成文件分离，避免大小写混用，避免单复数混用，避免中英混用
html文件中禁止出现行内样式，不推荐页内样式，页内样式尽快提取到css文件中
html文件中的自定义属性前需加data-前缀，以区别于html的原生属性
html保持结构简洁明了、协调一致、富含语义和可扩展性，尽量少引入装饰性的图片
css基于css3标准去写，不考虑不支持css3的浏览器，私有前缀通过插件去加
css样式命名参照bootstrap，保持兼容和方便上手
js基于es5、es6去使用方法，不支持的方法做polyfill
除非必须不要修改第三方库的源码以及，应在外部打补丁，方便后续升级

css文件组织

css文件组织。
global.css 页面的组件样式库。
item.css 根据项目需求，修改的特殊样式或定义更高级的样式。（放到最后）
index.css 首页样式
login.css 登录页样式
admin.css 后台样式
helpers.scss 辅助样式
reset.scss 重置样式
mixin.scss 定义sass的变量

js文件组织

common.js 自定义的一些方法，插件的公共调用等。
widget.js 定义的插件，与组件相关的方法 复用性高的方法等。
utils.js 辅助函数，项目无关。
is.js 各种校验 http://is.js.org/
jquery.js dom操作库，注意版本号
underscore.js 数组集合对象的函数式操作库 http://www.bootcss.com/p/underscore/
moment.js 日期操作库 http://momentjs.cn/
三方库以文件夹的形式可移入scripts文件夹，超大库如arcgis可独立文件夹，数据文件放入data文件夹

html文件组织

html代码公用部分需预留钩子 方便整体替换
html文件名用-分割单词 避免出现各种括号空格
html文件中避免页内样式 少写页面脚本
通用的组件调用要写到common.js中
当前页面的高亮样式用js去实现判断
echarts以及其他组件的通用配置部分提取出公用
减少页面中的全局变量，可以挂载到命名空间下
非首屏需要js、css文件移到body结束之前

img文件组织

* 图标图片加 icon- 前缀，且放置在images/icon文件夹中
* 背景图片加 bg- 前缀
* 按钮图片加 btn- 前缀
* 内容图片加 img- 前缀
* 相关图片通过命名处于相邻位置
* 图片的状态放在文件名的最后，如btn-delete-on.png
* 状态名称准确命名，如on/off/active/hover/disabled/focused
* 多从html和css的属性中去找单词，而不是自己生造

代码规范

参考Code Guide by @AlloyTeam

* 缩进：html标签的子标签不缩进，即head、body和html对齐，head、body的子标签要缩进，影响范围大务必遵守
* 缩进：使用soft tab（4个空格），1个tab=4个空格，将tab转为空格，缩进只有空格，保持各编辑器中格式统一
* 空白：head和body之间空一行，html、css、js文件中连续空行不要多于一行，无意义
* 符号：html中属性值用双引号（""）包围，js中的字符串用单引号（''）包围，对象中的键非必须不要加引号
* 符号：html中script和style标签的内容缩进，css属性末尾都要加分号，js要加分号
* 结构：结构保持通用，少用JS生成，减少数量层级，事件、循环、遍历的形参要统一
* 推荐使用Sublime Text的HTML-CSS-JS Prettify插件去格式化
* 注释要恰到好处，既说明问题，又不显啰嗦，不能成垃圾代码

html标签属性顺序

  • class
  • id
  • name
  • data-*
  • src, for, type, href, value , max-length, max, min, pattern
  • placeholder, title, alt
  • aria-*, role
  • required, readonly, disabled
* class是为高可复用组件设计的，所以应处在第一位；
* id更加具体且应该尽量少使用，所以将它放在第二位。
* boolean属性指不需要声明取值的属性，WhatWG section on boolean attributes。boolean属性的存在表示取值为true，不存在则表示取值为false。

css属性顺序

  • 1.位置属性(position, top, right, z-index, display, float等)
  • 2.大小(width, height, padding, margin)
  • 3.文字系列(font, line-height, letter-spacing, color- text-align等)
  • 4.背景(background, border等)
  • 5.其他(animation, transition等)
* 有相应插件如理，如csscomb
